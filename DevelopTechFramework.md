# 汪汪机开发技术框架

## 技术架构概览

**后端核心**：C++

**前端UI**：Flutter

**支持平台**：iOS、Android

**通信方案**：Flutter FFI（Foreign Function Interface）

**选择理由**：
- 直接调用 C++ 函数，性能最优
- 无消息传递开销
- Flutter 官方支持

## C++ 核心层

### 职责范围
- 跨平台核心业务逻辑
- AI接口对接与通信
- 数据存储与管理
- 角色人设系统
- 消息处理引擎

### 技术选型

**HTTP 库**：libcurl

**选择理由**：
- 性能最优，社区最成熟
- 经过 20+ 年验证，被 Chrome、Firefox 等大厂使用
- 支持 HTTP/2、HTTP/3
- 跨平台支持最好

**JSON 库**：RapidJSON

**选择理由**：
- 性能最优，适合高频数据交互
- 内存效率高
- 腾讯开源，稳定可靠

**数据库**：SQLite

**选择理由**：
- 最成熟的嵌入式数据库
- 零配置，单文件数据库
- 被广泛使用，稳定可靠
- 支持完整 SQL 语法

**日志库**：spdlog

**选择理由**：
- 超高性能，最快的 C++ 日志库之一
- 社区非常活跃，GitHub 20k+ stars
- 头文件库，集成简单
- 支持异步日志、丰富的输出目标

## Flutter UI层

### 职责范围
- 用户界面渲染
- 用户交互处理
- 状态管理
- 平台适配

### 技术选型

**状态管理**：Riverpod

**选择理由**：
- Provider 的进化版，性能优秀
- 编译时安全，减少运行时错误
- 不依赖 BuildContext，使用更灵活
- 支持细粒度更新，高性能
- 有完善的代码生成工具

**网络请求**：dio

**选择理由**：
- 性能最优，专为 Flutter 优化
- 社区最成熟，被大量生产环境使用
- 功能全面，支持拦截器、重试、超时等
- 错误处理完善

**本地存储**：无（所有数据通过 FFI 交由 C++ SQLite 处理）

**架构说明**：
- 所有核心数据由 C++ 层 SQLite 统一管理
- Flutter 通过 FFI 调用 C++ 接口读写数据
- 避免数据分散，保持架构简洁

**日志库**：logging + logger 组合

**选择理由**：
- logging 是 Dart 官方包，最稳定
- logger 输出美观，开发体验好
- 两者结合：logging 用于生产，logger 用于开发

**构建系统**：CMake

**选择理由**：
- 最成熟稳定，社区庞大
- 跨平台支持最好
- Flutter 官方推荐

## CI/CD流程

**iOS 打包方案**：GitHub Actions

**流程说明**：
- 由于开发者无 macOS 环境，iOS IPA 包完全使用 GitHub Actions 编译
- 打包未签名的 IPA 包
- 下载后通过社群分发

**Android 打包方案**：本地编译

## 测试框架

**C++ 测试**：Google Test

**选择理由**：
- C++ 行业标准测试框架
- 功能全面、断言丰富
- 与 CMake 集成好

**Flutter 测试**：flutter_test + mockito

**选择理由**：
- flutter_test：Flutter 官方测试框架
- mockito：支持 Mock 对象，用于单元测试
